# Native Acceptance Protocol (retired)

Last reviewed: 2026-07-10

**This document has been superseded by the [Release Gate Checklist](../engineering/Release_Gate_Checklist.md).**

It dated from Phase 7 and described an early mix of automated E2E notes and
manual OS checks; its checked-off sign-off boxes no longer reflected any real
release run and contradicted the (then-empty) release checklist. Everything
still relevant — the axe/a11y standards enforced by the Playwright suites, the
recovery drills, and the manual OS-level checks (trash, watcher, focus traps,
keychain, printing) — now lives in the Release Gate Checklist, which is filled
in as a copy for every release.

For how the automated suites work day to day, see `CONTRIBUTING.md` (tests and
definition of done) and `docs/engineering/Performance_Notes.md` (benchmarks).
