# セキュリティと共有

## セキュリティセンター、再暗号化、公開 Slice

ダッシュボードはモックアップの復旧・デバイス・チームカードに対応し、必要に応じて Vault、接続、設定、ロック解除を開きます。取り消し時は再開可能な完全再暗号化を実行できます。Vault Slice は **詳細 → 内容 → 権限 → 確認** で作成します。外部公開は別の暗号化 Workspace を使い、净化投影は私有属性・除外リンク・埋め込みを削除します。公開リリースには独立暗号レビューと実機 Android/iOS 証拠が必要です。

最終確認: 2026-07-22

Plainvaはデバイス上の保管庫を通常の読めるファイルのまま保ち、クラウドコピーを不透明な暗号化オブジェクトとして保存します。アカウント接続後、**設定 → 保管庫 → セキュリティと共有**を開きます。

## 設定

1. 所有者名とデバイス名を選びます。鍵はOSキーチェーン、利用できない場合はローカルパスフレーズで保護されます。
2. `.pvrecovery`ファイルを保存し、表示された復旧コードを別の場所に保管します。各コードブロックには見やすいグループ番号があります。強調表示された2グループの値を入力し、バックアップを読めることを確認します。復旧には両方が必要で、クラウド認証情報は含みません。
3. ワークスペースを有効化します。Plainvaは署名済みポリシーを公開し、全ファイルを`.pvws/`へ暗号化します。ローカル保管庫は読めるままで、移行は中断後に再開します。

旧平文は移行中`.pvws/`の隣に残ります。状態が**保護済み**になってから明示的に削除でき、ローカルファイルは削除されません。

オフライン変更は永続キューに残ります。削除には署名済みtombstoneが必要で、並行変更は`.CONFLICT-…`コピーとして保持されます。

## デバイスと復旧

新しいモバイル端末はQR／コード要求を作成します。承認済みのデスクトップで短いコードを入力し、両方のフィンガープリントを比較してから承認してください。削除した端末は新しい変更に署名できません。全端末を失った場合、**アクセスを復旧**で`.pvrecovery`ファイルと別保管のコードを使い、内容を書き換えずに新しい所有者端末を作成します。**復旧情報を更新**は二重署名アンカーで新しいIDを登録し、古いセットを無効にします。

## メンバー、ロール、Slice

所有者と管理者はメンバーを招待し、グループを作成し、ロールをワークスペース全体、Slice、または1オブジェクトに限定できます。Editorは編集、Commenterはコメント、Readerは読み取りのみ、Contributorは指定範囲で新規作成のみ可能です。ローカル書き込み前と署名前の両方で検査され、インポート、復元、自動化、AI操作にも適用されます。

Sliceはフォルダー、明示選択、またはパス・種類・タグ・プロパティの動的ルールです。公開前に必ず**プレビュー**を使ってください。権限のないオブジェクトは実体化されず、検索、グラフ、プレビューにも入りません。

## コメント、バージョン、隔離

コメントと解決マーカーも暗号化・署名されます。**バージョン履歴**は暗号化リビジョンを読み、過去版を新しい署名変更またはコピーとして復元します。不正なリモート成果物は**整合性とローカルfork**に隔離され、再試行、ciphertextの書き出し、修復済み、無視を選べます。他の同期を止めず、リモートに存在しないだけでは削除になりません。

## 暗号化された保管庫を正しく削除する

暗号化された保管庫が不要になったら、クラウドフォルダーを削除する**前に**Plainvaで廃止してください。順序が重要です。フェイルクローズドの保護は、Plainvaがまだ接続を暗号化されたものと想定しているあいだにクラウドコピーが消えると、同期を停止したままにします — これは、攻撃者が暗号化を剥がして平文を強制するのを防ぎます。

1. **設定 → 保管庫 → Security & Sharing** を開きます。
2. 復旧カードで **ワークスペースを廃止** を選びます。Plainvaはこの端末のローカル鍵とワークスペースデータを消去し、保管庫を通常の保管庫として開き直します。
3. クラウドから消したい場合は、そのうえで初めてプロバイダー側のクラウドフォルダー（`.pvws/` オブジェクト）を削除します。Plainvaが暗号化されたクラウドオブジェクトを代わりに削除することはありません。

すでにクラウドコピーを削除していて、同期が「ワークスペースがありません」または「マニフェストがありません」というエラーで失敗する場合、対処はエラーが表示される場所で提供される同じリセットです。

- 暗号化された**ワークスペース**の場合は **Security & Sharing** を開きます。ステータスに復旧の注記付きのエラーが表示されるので、**ワークスペースを廃止** を選び、この端末のワークスペースをリセットすると同期が再び動作します。
- 内容が暗号化された**同期接続**の場合は、同期ステータスをクリックして同期エラーダイアログを開き、**暗号化をリセット** を選びます。このボタンは、リモートの暗号化データが欠落しているか無効な場合にのみ表示されます。

どちらの操作も明示的で、確認を伴います。Plainvaが暗号化された接続を黙って平文に格下げすることはなく、どちらの操作もローカルファイルを削除しません。クラウドに本当に必要な暗号化コンテンツがまだある場合は、代わりにキャンセルしてください — リセットすると平文同期が再開されます。

**アプリデータを消去**で保管庫を削除する（Splash → 保管庫を削除 → アプリデータも消去）と、これらの暗号化マーカーも消えます。そうして削除した保管庫は、あとの再接続を妨げるものを何も残しません。
