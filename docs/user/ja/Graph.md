# グラフ

最終更新: 2026-07-14

Plainvaのグラフは、ポスターではなく作業のためのツールです。自分がどこにいるか、何がつながっているか、何が欠けているかを示し、その場で直接操作できます。グラフエンジンは1つだけで、3つの顔を持っています。

## コンテキストグラフ（右サイドバー）

右サイドバーの**グラフ**セクションを開きます。中央にアクティブなノート、その上にフォルダー構造、フォルダー概要（index.md）の場合はその下に含まれるノート、左側に受信参照、右側に発信参照が表示されます。データベースからのリレーションは、そのプロパティ名をラベルとして表示します。

- ノードをクリックするとそのノートが開きます（フォーカスは一緒に移動します）。
- Ctrl/Cmd+クリックで分割表示、中クリックで新しいタブで開きます。
- ノードを別の場所にドラッグするとその場に固定されます（小さなドット表示）。この配置はノートごとに記憶されるため、次回そのノートを開いたときも同じ配置で表示されます。アクティブなノートは常に中央に表示されます。右上の**固定ピン**が記憶のオン・オフを切り替えます。オフにすると、このノートの記憶された配置は破棄されます。
- その下には、最大3件の**提案**が表示されます: アクティブなノートに言及しているがリンクされていないノート、よく一緒にリンクされるノート、似た近傍を共有するノート、まれなタグを共有するノートです。提案されたノートのタイトルがアクティブなノートにテキストとして含まれている場合、提案には**リンクされる箇所のプレビュー**が表示されます。**リンク**はその箇所をそのままWikiリンクに変換します（表示テキストがリンク先と異なる場合は`[[リンク先|テキスト]]`の形式になります）。一致する箇所がない場合は、ノートの末尾にリンクが追加されます（プレビューにその旨が表示されます）。**提案を却下**は、あなたの判断を記憶します。

## 保管庫マップ（独立したタブ）

**Ctrl/Cmd+Shift+G**、一番左の**アクションバー**にあるグラフアイコン、またはコマンドパレット（**グラフを開く**）からマップを開きます。独立したタブで開きます。もつれた糸くずの代わりに、実際のフォルダー構造がバブルとして表示されます——バブルをダブルクリックするとそのノートが展開され、**すべてのフォルダーを折りたたむ**で元に戻ります。レイアウトは決定的です: 同じマップは、開くたびに毎回同じ見た目になります。**マップの移動**は中クリックまたはCtrl/Cmd+ドラッグ、**ズーム**はマウスホイールで行います。ノードをドラッグするとその場に固定されます（小さなドット表示）。右上の**固定ピン**が記憶のオン・オフを切り替えます: オフにすると、このマップの記憶された配置は破棄され、自動レイアウトに戻ります（右クリックメニューの**レイアウトをリセット**と同じです）。固定はデバイスごとに保存されます。

ヘッダーバーのツール:

- エッジのスタイルが一目でわかります（凡例、左下）: **リレーション**はラベル付きの実線（アクセントカラー）、**リンク**は破線、**埋め込み**は点線です。
- **検索**は一致しないものをすべて薄く表示します。**タイプ**（OKF）と**タグ**で絞り込めます。エッジの種類（**リンク**、**リレーション**、**埋め込み**）は個別に切り替えられます。
- Plainvaが管理する概要ノート（`index.md`と`log.md`）は、ほぼすべてにリンクしてグラフを見づらくするため、デフォルトでは非表示になります。これはコンテキストグラフとデータベースグラフにも当てはまります。保管庫マップでは、**フィルター**ボタンの**index.md を表示**チェックボックスで再表示できます。
- **選択にフォーカス**は、選択したノートとその近傍1〜3ホップだけにマップを絞り込みます。
- **ヒートマップ**は、最近編集されたノート（7日/30日/90日）を明るく表示します——「自分は何に取り組んでいたのか？」がわかります。
- **タイムトラベル**は、作成日順にノートを表示します。スライダーで保管庫の成長を再生できます。日付は`date`/`datum`プロパティから取得され、なければファイルの作成日から取得されます（クラウド専用の保管庫では近似値になります）。

マップ上での操作:

- あるノードを別のノードの**上に**ドラッグする: Plainvaはテキストリンクの作成を提案します——または、データベースに一致する**リレーション**があれば、それを直接提案します（リレーションが1件のみを許可している場合、Plainvaは置き換える前に確認します）。
- ノードを右クリック: 開く、ピーク、分割で開く、**接続された新規ノート**、名前を変更（保管庫全体でリンクを更新）、ブックマーク、削除。
- 空白部分を右クリック: **新規ノート**、レイアウトをリセット、**PNG/SVGとしてエクスポート**。
- フォルダー間のエッジの束をクリックすると、個々のリンクが一覧表示されます。エッジにカーソルを合わせると、そのリンクが含まれる文が表示されます。
- **空白部分でのドラッグ**は選択範囲の矩形を描き、複数のノートをマークします（Shift+ドラッグで既存の選択範囲を拡張します）。その後マークされたノードの1つをドラッグすると、すべてが一緒に移動します。フッターには選択範囲に対するブックマーク/削除が用意されています。

## クリーンアップ

**クリーンアップ**ボタンを押すと、3つのタブを持つ作業リストが開きます: **孤立ノート**（接続のないノート）、**壊れたリンク**（存在しないリンク先——**ノートを作成**で作成できます）、**言及**（**保管庫をスキャン**で、ノートが名前で言及されているがリンクされていない箇所を検出します。**リンク**でその箇所をWikiリンクに変換します）。マップのフッターには孤立ノートの件数が表示され、クリックするとパネルが開きます。

## データベースビューとしてのグラフ

すべての`.base`データベースは**グラフ**ビューを追加できます（ビューを追加 → **グラフ**）: データベースの行がノードになり、あなたの**リレーション**がラベル付きのエッジになります。ヘッダーバーで、エッジにするプロパティ、選択プロパティによる**色分け**、数値プロパティによる**サイズ分け**、そして**外部の対象**（データベースの外を指すリレーション）または**受信リレーション**（他のデータベースからこれらのエントリを指すリレーション——例えばプロジェクトのタスク）を表示するかどうかを選べます。このビューはObsidian互換の形式で保存されます——Obsidianでは同じファイルがテーブルとして表示されます。

## 制限事項

- グラフはノート（ファイル）を表示するもので、個々の段落は表示しません。
- 固定と却下した提案は`.plainva/`配下に保存され、同期では移動しません——基本のレイアウトはすべてのデバイスで同一です。
- 提案は純粋な保管庫の分析結果です。あなたのマシンから外部に送信されるデータはありません。
