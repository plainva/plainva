# 同期の設定

最終更新: 2026-07-20

Plainvaは、各保管庫を任意で選択したストレージと同期できます——アプリから直接行われ、Plainvaが運用する仲介サービスは一切ありません: あなたのデータは、あなたのコンピューターとあなた自身のアカウント/サーバーの間だけを移動します。このページでは、プロバイダーごとの設定手順を説明します。

一般的にどのサービスが利用可能か（WebDAV経由やプロバイダーのデスクトップクライアント経由も含む）は、[同期の互換性](Sync_Compatibility.md)で説明しています。

## 基本

- 設定は**設定 → 保管庫 → クラウドアカウント**にあります: **アカウントを接続…**でアシスタントが開きます——まず**プロバイダー**を選び、次に**サービス**にチェックを入れ（ファイル同期の場合: **ファイル**）、その後ログインします。タイルの一覧はプロバイダーを実際の普及度順に並べています。**プロバイダーを検索…**では、プリセットとして用意されているメールプロバイダーも見つかります。**ファイル**サービスを担うアカウントは、保管庫ごとに常に**ちょうど1つ**です。その後、**同期**エリアに接続済みのアカウントと**クラウドフォルダー**が表示され、動作（**同期間隔**、キュー）を管理できます。**アカウントを管理**でクラウドアカウントに戻ります。
- **ファイル**サービスでは、**Microsoft**（OneDrive）、**Google**（Drive）、**Dropbox**、**Nextcloud**、**オブジェクトストレージ (S3)**、汎用の**WebDAV / CalDAV**に加えて、**Fastmail**、**mailbox.org**、**Yandex**、**Mail.ru**、**Koofr**、**pCloud**もタイルとして用意されています。これらでは、メールアドレスと**アプリパスワード**さえあれば十分です——サーバーアドレスはあらかじめ入力済みです（WebDAVベース。**詳細設定：エンドポイントを個別に指定**で変更できます）。
- **ウェルカム画面から既存のオンライン保管庫を開く**: **保管庫を開く** → **オンライン保管庫**では、プロバイダーが異なっても同じ3つの手順で案内されます——**1. 接続**（ログインまたは認証情報の入力）、**2. クラウド内のフォルダーを選択**（そこでも**新しいフォルダー**から新規にフォルダーを作成できます）、**3. ローカルフォルダーを選択または作成**。また、すでに開いている保管庫についても、設定からいつでも同期を設定できます。
- **クラウドに新しい保管庫を作成する**: **新しい保管庫** → **オンラインサービス上**——まず開始時の構造（空、またはPARAなどのテンプレート）を選び、続いて接続してクラウド上の対象フォルダーを選択するか**新しいフォルダー**で作成し、最後にローカルフォルダーを選びます。構造はローカルフォルダーに作成され、最初の同期で自動的にアップロードされます。
- ローカルでの保存は即座にアップロードされます。設定された**同期間隔（秒）**でPlainvaがリモートの変更を確認します。
- オフライン中の変更はキューに蓄積され、次に接続したときに転送されます。ステータスバーには**オンライン**/**オフライン**が表示され、同期インジケーターは状態を示します（クリックで**今すぐ同期**）。同期に時間がかかる場合や初回の同期では、ステータスバーに進捗が件数（例: **同期 123/540**）として表示されるため、保管庫全体を処理している様子を確認できます。
- オンライン保管庫を初めて接続すると、保管庫のサイズによっては初回の同期にしばらく時間がかかることを一度だけお知らせします——その間も作業を続けられます。
- 両方が同じファイルを変更した場合、Plainvaは自動的にそれらをマージします（3ウェイマージ）。それが不可能な場合、あなたのバージョンは`.CONFLICT`ファイルとして安全に保存されます——何も失われることはありません（[FAQ](FAQ.md)を参照）。
- **競合の解決**: 影響を受けたノートに表示されるバナー（および、ツリー内の`.CONFLICT`ファイルの右クリックメニューにある**競合を解決…**）から比較ダイアログが開きます——左側にファイルの現在の状態、右側にあなたの保存されたバージョンが表示され、ブロック単位で取り込みながら編集できます。**右側を保存して解決**は結果をファイルに書き込み、競合コピーを片付けます。**相手側を保持**はあなたのコピーを破棄します（バージョンのスナップショットは残ります）。同期エラーダイアログにも既存の競合コピーが一覧表示され、ワンクリックで同じ比較画面に移動できます。
- **大量削除からの保護**: 同期しているファイルのうち異常に多くのものが一度にクラウドで削除されようとしている場合（たとえばローカルの保管庫フォルダーが空にされたり移動されたりした場合）、Plainvaは削除を保留してまず確認します: **クラウドで削除**は削除を実行します。**削除しない（復元する）**は削除を破棄し、次回の同期時にファイルをクラウドから復元します。Plainva 内で自分で確認した削除は保留されません。大量削除（10 件を超えるファイル、または保管庫の 20% 超）の場合は、代わりに削除前にもう一度確認を求めます。
- 添付ファイル（画像など）も同期されます。
- **空のフォルダー**も同期されます: Plainvaで作成したフォルダーはすぐにクラウドに反映され、クラウド上の空のフォルダーも、遅くとも次回の完全な一覧取得までには他の端末に表示されます。
- 認証情報とトークンはオペレーティングシステムのキーチェーンに保存されます（状態: **設定 → アプリ → 情報と診断 → OSキーチェーン**）。保管庫内のファイルに保存されることはありません。
- **切断**は保管庫の同期を停止します。これによりどこかのファイルが削除されることはありません。

## WebDAV / Nextcloud

自前でホストするサーバーやほとんどのクラウドストレージにとって最もシンプルな方法です。

1. **クラウドアカウント** → **アカウントを接続…**で**Nextcloud**（または**WebDAV / CalDAV**）タイルを選びます。
2. **サーバーアドレス**、**ユーザー名**、**パスワードまたはアプリトークン**を入力します——可能な限り、メインのパスワードの代わりにアプリパスワードを使用してください（Nextcloudの場合: 設定 → セキュリティ → アプリパスワード）。
3. **接続**で認証情報を確認します。その後、**フォルダーを選択…**から**クラウドフォルダー**を選びます。

**Nextcloud**の特別な点: ファイル**と**カレンダーの両方が1つのフォームでまかなえます——PlainvaはサーバーアドレスからWebDAVとCalDAVのエンドポイントを自動的に導き出します（導き出されたアドレスはアシスタント内に表示されます。**詳細設定：エンドポイントを個別に指定**で別々のURLを設定することもできます）。両方のサービスにチェックを入れると、1回の手順で両方に接続できます。

一般的なサーバーアドレス（Nextcloud、Koofr、MagentaCLOUD、Storage Boxなど多数）は、[同期の互換性](Sync_Compatibility.md)に一覧があります。

## Google Drive

Google Driveは現在、自前の認証情報（「Bring Your Own」）で動作します: あなただけが所有する無料のGoogle Cloudプロジェクトを一度だけ作成します。手順は[Google Drive (BYO)](Google_Drive_BYO_Guide.md)を参照してください。

概要: **クラウドアカウント** → **アカウントを接続…**で**Google**タイルを選び、**ファイル**サービスにチェックを入れ、Googleプロジェクトの**クライアントID**と**クライアントシークレット**を入力し、**Googleでログイン…**をクリックします——ブラウザーでログインが開きます。接続後は、**フォルダーを選択…**からDrive内（サブフォルダーを含む、デフォルトは「Plainva」）の**クラウドフォルダー**を直接選べます。注意: Googleプロジェクトがテストモードの間、ログインは7日で期限切れになり、アカウントの詳細にある**再度ログイン**で更新する必要があります。

## OneDrive

Plainvaは独自のアプリ登録をあらかじめ用意しています——**自分でIDを用意する必要はもうありません**。

1. **クラウドアカウント** → **アカウントを接続…**で**Microsoft**タイルを選び、**ファイル**サービス（OneDrive）にチェックを入れます——希望すれば**カレンダーとタスク**や**メール**も同時に（1つのMicrosoftアカウントで3つすべてのサービスをまかなえます）。
2. **Microsoftでログイン…**をクリックし、ブラウザーでログインを確認します。これで完了です——Plainvaがフォルダー（デフォルトは「Plainva」）を作成し、外部から追加されたファイルも含めてその内容全体を同期します。
3. 任意: 接続後は、**フォルダーを選択…**からOneDrive内（サブフォルダーを含む）の**クラウドフォルダー**を直接選べます。

任意: **独自のアプリ ID を使用**から、自分で登録したクライアントIDを代わりに入力できます（例: 社内の制限がある場合）。詳細な手順ガイド: [OneDrive & Dropbox (BYO)](OneDrive_and_Dropbox_BYO_Guide.md)。

## Dropbox

Plainvaは独自のDropboxアプリをあらかじめ用意しています——**自前のアプリは不要です**。

1. **クラウドアカウント** → **アカウントを接続…**で**Dropbox**タイルを選びます（**ファイル**サービスのみを担います）。
2. **Dropboxでログイン…**をクリックし、ブラウザーで確認します。これで完了です（デフォルトフォルダーは`/Plainva`）。
3. 任意: 接続後は、**フォルダーを選択…**からDropbox内（サブフォルダーを含む）の**クラウドフォルダー**を直接選べます。

任意: **独自のアプリ ID を使用**から、自分で登録したアプリキーを代わりに入力できます。詳細な手順ガイド: [OneDrive & Dropbox (BYO)](OneDrive_and_Dropbox_BYO_Guide.md)。

## S3互換ストレージ

AWS S3、Cloudflare R2、Backblaze B2、MinIO、Wasabi、Hetznerなどに対応——キーベースで、ブラウザーログインは一切不要です。**クラウドアカウント** → **アカウントを接続…**で**オブジェクトストレージ (S3)**タイルを選び、フィールドを入力します:

| フィールド | 意味 |
|---|---|
| **エンドポイント** | S3 APIのベースURL。例: `https://s3.eu-central-1.amazonaws.com`、`https://<account>.r2.cloudflarestorage.com`、またはローカルMinIOの場合は`http://127.0.0.1:9000` |
| **バケット** | バケット名 |
| **リージョン** | SigV4リージョン。`us-east-1`はAWS以外のほとんどのストレージで動作し、Cloudflare R2は`auto`を使用します |
| **アクセスキーID** / **シークレットアクセスキー** | プロバイダーが発行するAPIキーペア |
| **キープレフィックス（任意）** | 保管庫を置くバケット内のサブフォルダーです。空欄 = バケットのルート |
| **パス形式URL** | 推奨設定です（MinIO、R2、ほとんどの互換ストレージ）。virtual-hosted形式のAWSバケットの場合のみ無効にしてください |

**キープレフィックス**（クラウドフォルダー）は、接続後に**フォルダーを選択…**からバケット内を直接選べます。

**接続**後、同期がすぐに開始されます。
