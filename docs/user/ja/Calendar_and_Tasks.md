# カレンダーと外部タスク

最終更新: 2026-07-18

Plainvaは、既存のカレンダーおよびタスクアカウント——**CalDAV**（Nextcloud、Fastmail、mailbox.orgなど）、**Google**（カレンダー + Tasks）、**Microsoft**（Outlookカレンダー + To Do）——を接続し、双方向で連携できます。あなたのノートが中心であり続けます: 予定は会議ノートになり、外部のタスクリストは通常のノートとして[既定のタスクデータベース](Tasks.md)に反映されます。

## アカウントを接続する

**設定 → 保管庫 → カレンダーとアカウント → アカウントを追加…**を開き、プロバイダーを選びます:

- **CalDAV**: サーバーURL、ユーザー名、そして**アプリパスワード**（Nextcloudの場合: 設定 → セキュリティ → デバイスとセッション）。登録もキーも不要です。
- **Google**: 独自のOAuthクライアントIDが必要です（Google Drive同期と同じBYOモデル——[Driveガイド](Google_Drive_BYO_Guide.md)を参照）。GoogleのCloudプロジェクトで、追加で*Google Calendar API*と*Google Tasks API*を有効にし、それらのスコープを同意画面に追加してください。ブラウザーが同意のために開きます。接続すると、何かを保存する前にアカウントが検証されます。
- **Microsoft**: **接続**をクリックしてブラウザーで確認するだけです——設定は不要です。

各アカウントには、その**カレンダー**（チェックを入れたものがカレンダータブに表示されます）と**タスクリスト**（デフォルトでは意図的にチェックが外されています——チェックを入れると、下記のタスク同期が始まります）が一覧表示されます。パスワードとトークンは、お使いのオペレーティングシステムのキーチェーンに保存されます。アカウントの下にある**会議フォルダー**の設定で、会議ノートの作成先を選べます。

## カレンダータブ

左のアクションバー（カレンダーアイコン）またはコマンドパレット（**カレンダーを開く**）から開きます。月グリッドに予定が表示され（カレンダーごとに色分けされたドット）、選択した日を一覧するデイペインが表示されます——まず終日の予定、続いて時刻・カレンダー名・場所付きの時間指定の予定です。ビューは数分ごとに自動的に更新されます。更新ボタンで強制的に更新することもできます。

- **新しい予定**: デイペインの**+**——タイトル、カレンダー、日付/時刻または終日の期間、場所、そして任意で簡単な**繰り返し**（毎日/毎週/毎月/毎年）を設定できます。
- **編集 / 削除**: 予定に付いている鉛筆とゴミ箱のアイコンです。変更は安全性チェック付きでプロバイダーに書き込まれます: その間に予定が外部で変更されていた場合、Plainvaは上書きする代わりに表示を更新します。
- **繰り返しの予定**には繰り返しバッジが付きます。1つのインスタンスを編集または削除すると、Plainvaは**「この予定のみ」**（例外を作成する、またはその回だけをスキップする）か**「すべての予定」**（シリーズ全体を変更する）かを尋ねます。既存の繰り返しルールをPlainvaが書き換えることは決してありません。

## 予定 → 会議ノート

どの予定にも付いているノートアイコンは、その**会議ノート**を作成（または再度開き）ます——会議フォルダー内の`YYYY-MM-DD タイトル.md`という名前の通常のノートで、日付・場所・参加者があらかじめ入力されており、フロントマターにはその予定と結びつける小さな`plainva.pim`マーカーが付きます。同じ予定を再度クリックすると、常に同じノートが開きます。たまたま同じ名前を持つあなた自身のノートが、勝手に扱われることは決してありません。

## タスクデータベース内の外部タスクリスト

接続済みのアカウントで**タスクリスト**にチェックを入れると、そのタスクがノートとして[既定のタスクデータベース](Tasks.md)に表示されます: タイトルがノート（H1）になり、期限は日付列に入り、完了はステータス列に反映されます（最初の選択肢 = 未完了、最後の選択肢 = 完了）。同期は双方向で、フィールドごとに行われます:

- ノート（タイトル、期限、ステータス）を編集する → 変更がプロバイダーへ送られます。
- タスクが外部で変更される → ノートが追従します。
- 両側が変更された場合、そのフィールドについてはローカルの変更が優先され、残りは外部の状態に従います。

2つの安全ルールがあなたのデータを守ります: **ノートを削除しても、プロバイダー側のタスクは決して削除されません**（同期が止まるだけで、再インポートもされません）。また、**プロバイダー側でタスクが削除されても、あなたのノートは決して削除されません**（単なる通常のノートになるだけです）。タスクノートの名前変更や移動は問題ありません——フロントマターのマーカーがリンクを保持します。

現在の制限: 通常のノートとして作成されたタスクはプロバイダーへプッシュされません（外部で作成するか、タスクデータベース経由で作成してください）。また、このページの内容はすべて、今のところデスクトップファーストです。
