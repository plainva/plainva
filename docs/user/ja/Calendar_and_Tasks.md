# カレンダーと外部タスク

最終更新: 2026-07-20

Plainvaは、既存のカレンダーおよびタスクアカウント——**CalDAV**（Nextcloud、Fastmail、mailbox.orgなど）、**Google**（カレンダー + Tasks）、**Microsoft**（Outlookカレンダー + To Do）——を接続し、双方向で連携できます。あなたのノートが中心であり続けます: 予定は会議ノートになり、外部のタスクリストは通常のノートとして[既定のタスクデータベース](Tasks.md)に反映されます。

## アカウントを接続する

**設定 → 保管庫 → クラウドアカウント → アカウントを接続…**を開き、プロバイダーを選び、サービスの選択ステップで**カレンダーとタスク**にチェックを入れます:

- **Nextcloud / CalDAV**: サーバーアドレス、ユーザー名、そして**アプリパスワード**（Nextcloudの場合: 設定 → セキュリティ → デバイスとセッション）。登録もキーも不要です——Nextcloudの場合、PlainvaはサーバーアドレスからCalDAVアドレスを自動的に導き出します（他のCalDAVサーバーの場合は**WebDAV / CalDAV**タイル、または**詳細設定：エンドポイントを個別に指定**を使用してください）。
- **Apple iCloud、Yahoo、AOL、Zoho、Fastmail、mailbox.org、Yandex、Mail.ru**: カレンダーアドレスがあらかじめ入力された専用タイルです——メールアドレスと**アプリパスワード**さえあれば十分で、サーバー欄は不要です（Appleの場合、アプリパスワードは必須です。アシスタントはプロバイダーのガイドにリンクします）。ご注意ください: Yahoo自身が、そのCalDAVサービスは不安定だとしています——動作がおかしい場合、それはPlainvaのせいではありません。
- **Google**: 独自のOAuthクライアントIDが必要です（Google Drive同期と同じBYOモデル——[Driveガイド](Google_Drive_BYO_Guide.md)を参照）。GoogleのCloudプロジェクトで、追加で*Google Calendar API*と*Google Tasks API*を有効にし、それらのスコープを同意画面に追加してください。ブラウザーが同意のために開きます。接続すると、何かを保存する前にアカウントが検証されます。
- **Microsoft**: **Microsoftでログイン…**をクリックしてブラウザーで確認するだけです——設定は不要です。1つのMicrosoftアカウントは、同じ手順で**ファイル**（OneDrive）と**メール**も担うことができます。

アシスタントはサービスごとの状態（「接続済み——n件のカレンダーが見つかりました」）を表示します。**カレンダー**（チェックを入れたものがカレンダータブに表示されます）と**タスクリスト**（デフォルトでは意図的にチェックが外されています——チェックを入れると、下記のタスク同期が始まります）の選択は、その後**カレンダー**エリアで管理します。そこには**会議フォルダー**（会議ノートの作成先）と**デフォルトのカレンダー**もあります。パスワードとトークンは、お使いのオペレーティングシステムのキーチェーンに保存されます。

## カレンダータブ

左のアクションバー（カレンダーアイコン）またはコマンドパレット（**カレンダーを開く**）から開きます。ヘッダーのスイッチで5つのビューを切り替えられます: **日**・**3日間**・**週**は、左側に時刻の目盛りが付いた**時間グリッド**を表示します。予定はその開始時刻にブロックとして配置され、高さは所要時間を表し、重なる予定は横に並んで表示され、赤い線が「現在時刻」を示します。終日の予定と（タスクの重ね表示がオンの場合は）期限付きのタスクは、グリッドの上にある帯状の領域に表示されます。**月**は、月グリッド（カレンダーごとに色分けされたドット）に加えて、右側に選択した日の1日分の時間グリッドを表示します。**アジェンダ**は、今後の数週間を日ごとにグループ化して一覧表示します。**今日**をクリックすると現在の日付に戻ります。矢印は表示中の期間（1日、3日間、1週間、または1か月）単位でページ送りします。週の最初の曜日は**週の開始日**の設定（設定 → アプリ → 外観: 月曜日、土曜日、または日曜日）に従います——これはサイドバーのカレンダーにも適用されます。ビューは数分ごとに自動的に更新されます。更新ボタンで強制的に更新することもできます。すでに終了した予定は（Googleカレンダーと同様に）**薄く**表示されるため、今日の残りの予定が目立ちます。

- **新しい予定**: **時間グリッドの空いている場所をクリック**すると、小さなクイック作成ポップオーバー（タイトル、時刻、カレンダー、場所）が開きます——**保存**ですぐに作成され、**その他のオプション**で完全な予定ダイアログが開きます。グリッド上を**ドラッグ**すると所要時間を設定できます。ヘッダーの**+**は完全なダイアログを開きます: タイトル、カレンダー、日付/時刻または終日の期間、場所、**説明**、**色**、**参加者**、そして任意でOutlookスタイルの**繰り返し**を設定できます。色は、その予定だけについてカレンダーの色を上書きします（Microsoftアカウントには影響しません——Outlookには予定ごとの色設定がありません）。
- **参加者**: メールアドレスを入力して **Enter** キー（またはカンマ）を押すと**チップ**として追加されます。×で1件削除できます。繰り返しは日付/時刻のすぐ隣で設定します——頻度、間隔、（毎週の場合は）曜日、そして終了条件（なし / 日付を指定 / N回後）を選べます。既存の予定に繰り返しを追加したり変更したりすることもできます。
- **編集 / 削除**: 時間グリッドで**予定をクリック**すると、その値があらかじめ入力されたダイアログが開き、**会議ノート**と**削除**のアクションが表示されます。変更は安全性チェック付きでプロバイダーに書き込まれます: その間に予定が外部で変更されていた場合、Plainvaは上書きする代わりに表示を更新します。
- **移動 / サイズ変更**: 時間グリッドで予定を直接**ドラッグ**して調整できます——本体をドラッグすると日時が変更され（週・3日間ビューでは別の日にまたがることもできます）、**下端**をドラッグすると所要時間が変わります。新しい時刻はすぐにプロバイダーに書き込まれます（繰り返しの予定は、今のところダイアログ経由でのみ編集できます）。
- **出欠回答**: 予定に招待されている場合、ダイアログから**承諾**、**仮承諾**としてマーク、または**辞退**を選べます——Plainvaはその回答をプロバイダー（Google/Microsoft/CalDAV）経由で送信します。**参加者一覧**には、誰が承諾または辞退したか（返信状況）が表示されます。
- **招待メール**: 予定に参加者がいる場合、**参加者にメールで通知**にチェックを入れます。Googleの場合、Plainvaはその予定についてGoogleネイティブの招待を送信するよう依頼します（同じ予定なので、相手からの返信はあなたの予定に同期して反映されます）。Microsoftは参加者に自動的に通知します。CalDAVの場合——または自分のメールボックスからコピーを送りたい場合——カレンダーの**メールで送信**アクションを使うと、標準準拠のiCalendar招待を添付したメール作成画面が開くので、Gmailなど他のメールクライアントでは予定として表示され、はい/たぶん/いいえで回答できます。
- **他のカレンダーをブロック**: 予定の**コピー**アクション（またはダイアログ内の**他のカレンダーをブロック**ボタン）を使うと、他の書き込み可能なカレンダーの1つ以上にその予定を反映できます——中身の見えない**予定あり**のプレースホルダーとして、または（Notion Calendarのように）**詳細付き**として反映するかを選べます。繰り返しの予定は繰り返し設定ごと反映されるため、ブロックも繰り返されます。
- **繰り返しの予定**には繰り返しバッジが付きます。1つのインスタンスを編集または削除すると、Plainvaは**「この予定のみ」**（例外を作成する、またはその回だけをスキップする）か**「すべての予定」**（シリーズ全体を変更する）かを尋ねます。既存の繰り返しルールをPlainvaが書き換えることは決してありません。
- **タスクを表示**（更新ボタンの隣にあり、既定のタスクデータベースが設定されている場合にのみ表示されます）: [既定のタスクデータベース](Tasks.md)の期限付きの項目を、時間グリッドの帯状の領域と月グリッドに重ねて表示します。完了したタスクには取り消し線が付きます。デフォルトではオフになっており、この選択はデバイスごとに記憶されます。

## 予定 → 会議ノート

どの予定にも付いているノートアイコンは、その**会議ノート**を作成（または再度開き）ます——会議フォルダー内の`YYYY-MM-DD タイトル.md`という名前の通常のノートで、日付・場所・参加者があらかじめ入力されており、フロントマターにはその予定と結びつける小さな`plainva.pim`マーカーが付きます。同じ予定を再度クリックすると、常に同じノートが開きます。たまたま同じ名前を持つあなた自身のノートが、勝手に扱われることは決してありません。

## タスクデータベース内の外部タスクリスト

接続済みのアカウントで**タスクリスト**にチェックを入れると、そのタスクがノートとして[既定のタスクデータベース](Tasks.md)に表示されます: タイトルがノート（H1）になり、期限は日付列に入り、完了はデータベースの**完了チェックボックスプロパティ**に反映されます（ステータス列はそれに追従します。チェックボックス列を持たないデータベースは、ステータスの慣習——最初の選択肢 = 未完了、最後 = 完了——を使用します）。同期は双方向で、フィールドごとに行われます:

- ノート（タイトル、期限、ステータス）を編集する → 変更がプロバイダーへ送られます。
- タスクが外部で変更される → ノートが追従します。
- 両側が変更された場合、そのフィールドについてはローカルの変更が優先され、残りは外部の状態に従います。

2つの安全ルールがあなたのデータを守ります: **ノートを削除しても、プロバイダー側のタスクは決して削除されません**（同期が止まるだけで、再インポートもされません）。また、**プロバイダー側でタスクが削除されても、あなたのノートは決して削除されません**（単なる通常のノートになるだけです）。タスクノートの名前変更や移動は問題ありません——フロントマターのマーカーがリンクを保持します。

現在の制限: 通常のノートとして作成されたタスクはプロバイダーへプッシュされません（外部で作成するか、タスクデータベース経由で作成してください）。また、このページの内容はすべて、今のところデスクトップファーストです。
