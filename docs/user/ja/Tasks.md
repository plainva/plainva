# タスク

最終更新: 2026-07-17

タスクビューは、保管庫内のすべてのチェックボックスを1か所に集約します: すべてのノートにまたがる`- [ ]`と`- [x]`のリスト項目を、それらが属するノートごとにグループ化して表示します。これはプレーンなMarkdownの上に成り立つ「まだ何をやらなければならないか？」というビューです——プラグインも特別なファイルも不要です。

## 専用ビューがある理由（`.base`ではない理由）

[データベース（`.base`）](Databases_Base.md)はノート全体を対象に動作します——1ノートにつき1行です。チェックボックスはノート内の1つの*行*であり、1つのノートに多数のチェックボックスが含まれることもあるため、`.base`ではそれらを一覧表示できません。タスクビューは行ベースです: タスクの行を直接読み取るため、10個のサブタスクを持つ1つのプロジェクトノートがあれば、その10個すべてが表示されます。

## タスクビューを開く

- 左端のアクションバーにある**チェックリストアイコン**をクリックするか、
- **コマンドパレット**（`Ctrl/Cmd+P`）を開いて**タスクを開く**を実行します。

通常のノートと同様に、タブとして開きます。

## リストの読み方

タスクはノートごとにグループ化されます。ノートのタイトルは見出しになっており、クリックするとそのノートが開きます。各タスクにはそのチェックボックスとテキストが表示され、完了すると取り消し線が引かれます。タスクの行に`📅 2026-08-01`のように書かれた**期限**は、小さなバッジとして表示されます。

## フィルタリング

上部のバーでリストを絞り込みます:

- **未完了 / 完了 / すべて** — チェックボックスの状態による絞り込み（初期状態は**未完了**）。
- **タスクをフィルター…** — フリーテキストで、タスクのテキストに一致するものを絞り込みます。
- **すべてのフォルダー** — 選択したフォルダー（およびそのサブフォルダー）内のタスクのみを表示します。
- **すべてのタグ** — 選択したインラインの`#tag`を持つタスクのみを表示します。
- **期限あり** — `📅`の日付を持つタスクのみを表示します。

タグと期限は、タスクの行から直接読み取られます——例: `- [ ] Pay invoice #finance 📅 2026-08-01`。

## タスクを完了にする

タスクの**チェックボックス**をクリックすると、未完了と完了の間で切り替わります。変更はそのままノートに書き戻されます（通常の安全なファイル書き込みとして——`[ ]`/`[x]`の1文字だけが変わります）。そのため、ノート、Obsidian、そして同期はすべて整合性を保ちます。タスクの**テキスト**をクリックすると、代わりにノートが開き、その行までジャンプします。

リストが作成された後にノートが変更された場合、古くなった切り替え操作はスキップされ、リストが再読み込みされます——右上の**更新**ボタンでいつでも再読み込みできます。

## 既定のタスクデータベース

チェックボックスはさっと書き留めるのに便利ですが、時には行が育って、ステータスや期限、専用のノートを持つ「本物の」タスクになることもあります。そのためには、**設定 → 保管庫 → コンテンツと構造**で**既定のタスクデータベース**を選びます——そうしたタスクが独自のノートとして存在する[データベース（`.base`）](Databases_Base.md)です。**新規データベースを作成…**は、すぐに使える状態のものを組み立てます（保存先フォルダーに加えて、ステータス列、期限列、テーブルビュー、ボードビューを備えた`.base`）。既存のデータベースを選ぶこともできます。

設定すると、タスクビューは2つのセクションを表示します: 上に**タスクデータベース**のエントリー（ステータスと期限付き。**データベースとして開く**でボードやフィルターを含む完全なデータベースビューにジャンプします）、下に**ノートから**——おなじみのチェックボックスの一覧です。

## チェックボックスをデータベースタスクに変える

各タスクの行には、データベースアイコンが付いています: **タスクデータベースへ移動**。1回クリックすると:

- データベースの保存先フォルダーに新しいノートを作成します（デフォルトのテンプレートが設定されている場合はそれを使用します）。
- `📅`の日付を期限列に反映し、未完了タスクには最初のステータスの選択肢を設定し、その行の`#tags`をノートのタグとして保存します。
- `source`プロパティを介して、新しいノートをソースノートへリンクし戻します。
- ソースノートのチェックボックス行を、新しいタスクノートへのWikiリンクに置き換えます——項目は書かれた場所でそのまま読めるままで、タスクは今やデータベースの中に存在します。

アイコンを**右クリック**すると、代わりに別のデータベースを対象として選べます。既定のデータベースが設定されていない場合は、クリックするとすぐにそのピッカーが開きます。すべてプレーンなMarkdownのままです: 新しいタスクはフロントマターを持つ通常のノートであり、ソースノート内のリンクは通常の`[[Wikiリンク]]`です。

## タスクビューからノートを隠す

一部のノートに含まれるチェックボックスは、決して「本物の」タスクではありません——なかでも**テンプレート**がそうです。これらをリストから除外するために、ノート自身が自らを除外できます。真実はファイルの中にとどまります: この除外はノートのフロントマターフィールドであり、隠されたアプリの設定ではありません。同期され、Obsidianでも見え、どのテキストエディターでも確認できます:

```yaml
---
plainva:
  tasks: false
---
```

このフィールドを手で書く必要はありません:

- **タスクから隠す** — 各ノートのヘッダー行の右側に目のアイコンがあります。1回クリックすると、そのノートにマーカーが書き込まれ、非表示になります。
- **非表示を表示** — フィルターバーのこのオプションを選ぶと、非表示のノートが（薄く表示された状態で）リストに戻ります。それぞれに、**タスクに再表示**するためのアイコンが付いています（クリックするとマーカーが削除されます）。
- **テンプレートを隠す** — テンプレートフォルダーにチェックボックスを含むノートがある場合、右上に**テンプレートを隠す**ボタンが表示され、それらすべてに一度にマーカーを付与します。

新しく作成されたテンプレートには、自動的にマーカーが付きます。テンプレート**から**ノートを作成すると、マーカーは再び削除されます——新しいノートは実際のコンテンツであり、そのタスクは通常どおり表示されます。

## Obsidianとの互換性

タスクは通常のGFM（GitHub Flavored Markdown）のチェックボックスです。Plainvaが特別な構文を追加することはありません: 同じ`- [ ]`の行がObsidianでもチェックボックスとして表示され、どのエディターでも問題なく読み込めます。`📅 date`と`#tag`の慣習は一般的なObsidian-Tasksのスタイルですが、あなたのノートの中ではただのテキストです。
