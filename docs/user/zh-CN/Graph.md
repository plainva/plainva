# 关系图

更新日期：2026-07-09

Plainva的关系图是一个工作工具，而不是一张海报：它会告诉你自己身处何处、什么内容与之相连、又缺少了什么——而且你可以直接对此采取行动。这里只有**一个**关系图引擎，呈现为三种形态。

## 上下文关系图（右侧边栏）

打开右侧边栏中的**关系图**分区。它会在中间显示当前笔记，上方是文件夹结构；对于文件夹概览（index.md），下方会显示其中包含的笔记，左侧是传入引用，右侧是传出引用。来自数据库的关联会以其属性名称作为标签。

- 点击一个节点会打开该笔记（焦点会随之移动）。
- `Ctrl/Cmd`+点击在拆分中打开，鼠标中键点击在新标签页中打开。
- 拖动一个节点到其他位置，它会固定在那里（一个小圆点），并按笔记分别记住——重新打开该笔记时，你的排列会恢复原样。当前笔记始终保持在中间。
- 下方最多会出现三条**建议**：提及了当前笔记但未链接它的笔记、经常与它一起被链接的笔记、拥有相似邻域的笔记，或共享一个少见标签的笔记。如果标题以文字形式出现在正在编辑的笔记中，建议会显示将被链接的那段文字的预览；**链接**会把这段文字准确地变成一个Wiki链接（当显示文本与目标不同时，采用`[[目标|文本]]`这样的别名形式）。如果没有匹配的文字，链接会被添加到笔记末尾（预览会说明这一点）。**忽略建议**会记住你的决定。

## 仓库地图（独立标签页）

通过**Ctrl/Cmd+Shift+G**、最左侧**操作栏**中的关系图图标，或命令面板（**打开关系图**）打开地图。它会在独立的标签页中打开。你看到的不是一团乱麻，而是你真实的文件夹结构呈现为气泡——双击一个气泡展开其中的笔记，**折叠所有文件夹**则返回。布局是确定性的：每次打开时，同一张地图看起来都一样。拖动一个节点会将其固定（一个小圆点）；**重置布局**会清除所有固定。固定信息按设备存储。

标题栏中的工具：

- 一目了然的边样式（图例，左下角）：**关联**是带标签的实心强调色线条，**链接**是虚线，**嵌入**是点线。
- **搜索**会淡化所有不匹配的内容。可按**类型**（OKF）和**标签**筛选；边的种类（**链接**、**关联**、**嵌入**）可单独切换。
- **聚焦所选**会把地图缩小为一条选中的笔记加上1-3跳的邻域。
- **热力图**会高亮最近编辑过的笔记（7/30/90天）——"我之前在做什么来着？"
- **时间回放**按笔记的创建日期显示笔记；滑块会重放你仓库的成长过程。日期来自`date`/`datum`属性，否则来自文件创建日期（对于仅存在于云端的仓库，这是一种近似值）。

在地图上工作：

- 把一个节点拖到另一个节点**之上**：Plainva会提议写入一个文本链接——或者，如果来自你的数据库的某个**关联**恰好匹配（如果该关联只允许恰好一条记录，Plainva会在替换前先询问你），则直接写入该关联。
- 右键点击一个节点：打开、速览、在拆分中打开、**新建关联笔记**、重命名（会更新整个仓库中的链接）、添加书签、删除。
- 右键点击空白处：**新建笔记**、重置布局、**导出为PNG/SVG**。
- 点击文件夹之间的一束边，会列出其中的各条链接；将鼠标悬停在一条边上会显示该链接所在的那句话。
- `Shift`+拖动可选中多条笔记（页脚：为所选内容添加书签/删除所选内容）。

## 清理

**清理**按钮会打开一个待办列表，包含三个标签页：**孤立笔记**（没有任何连接的笔记）、**断开的链接**（目标不存在的链接——**创建笔记**可以创建它们）和**提及**（**扫描仓库**会找出某条笔记被提到但未被链接的地方；**链接**会把该处提及变成一个Wiki链接）。地图页脚会显示孤立笔记的数量——点击它会打开该面板。

## 作为数据库视图的关系图

每个`.base`数据库都可以拥有一个**关系图**视图（添加视图 → **关系图**）：数据库的行会变成节点，你的**关联**会变成带标签的边。在标题栏中，你可以选择作为边的属性、按某个选择属性**按颜色分类**、按某个数字属性**按大小分类**，以及是否显示**外部目标**（指向该数据库之外的关联）或**传入关联**（来自其他数据库、指向这些条目的关联——例如某个项目的任务）。该视图以兼容Obsidian的方式保存——Obsidian会把同一个文件显示为一张表格。

## 局限

- 关系图显示的是笔记（文件），而不是单个段落。
- 固定信息和被忽略的建议保存在`.plainva/`下，不会随同步传输——因此在每台设备上的基础布局都是一样的。
- 建议纯粹是仓库本身的分析结果；不会有任何内容离开你的电脑。
