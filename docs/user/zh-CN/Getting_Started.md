# 快速上手

更新日期：2026-07-22

本页带你从安装到第一次真正的工作：打开或新建仓库、认识界面，以及理解三种编辑器模式。

## 什么是仓库？

仓库就是电脑上一个存放Markdown笔记的普通文件夹。Plainva会添加一个隐藏的`.plainva/`子文件夹，用来存放搜索索引和设置——你的笔记本身仍然是未被改动的`.md`文件。你可以拥有多个仓库（例如"个人"和"工作"），并在它们之间切换。

## 打开或新建仓库

启动时，欢迎界面会显示：

- **打开仓库**——Plainva会先问"你的仓库在哪里？"：**本地文件夹**用于打开这台电脑上一个已有的Markdown文件夹（Obsidian仓库可以直接使用）；**在线仓库**会把云端已有的仓库同步到一个本地文件夹——每个服务商都遵循相同的三个步骤（**连接**、**选择云端文件夹**、**选择或创建本地文件夹**；参见[设置同步](Sync_Setup.md)）。
- **新建仓库**——首先会问"你的仓库要放在哪里？"（**在这台电脑上**或**在在线服务中**），然后选择起始结构：从空白开始，或使用预设的文件夹结构；两者随时都可以调整。**空仓库**只包含一个`index.md`概览。可用的模板有：**PARA**、**Zettelkasten**、**ACE (Linking Your Thinking)**、**Johnny.Decimal**、**GTD**和**Journal**——每一种都会创建文件夹、一篇带快速指南的欢迎笔记，以及以[OKF格式](OKF.md)自动维护的`index.md`概览（文件夹和文件名称跟随应用语言）。**Journal**模板还会额外配置好仓库的日记设置。**PARA**、**GTD**、**Zettelkasten**和**Journal**模板还会附带已经关联好的[数据库](Databases_Base.md)以及配套的笔记模板——例如带有状态看板和领域关联的项目，或指向所属项目的任务。在线路径下，连接会紧接着结构进行：选择服务商、连接、选择云端文件夹或通过**新建文件夹**新建一个，再选择本地文件夹——所选结构会被创建在本地文件夹中，并在首次同步时上传到云端。

**最近使用的仓库**列出了你之前打开过的所有仓库。**从列表中移除**只会从Plainva中移除该条目——文件仍保留在磁盘上。启用**启动时自动打开上次的仓库**可以在以后跳过欢迎界面。移除时，Plainva 会询问是否同时清除该仓库的所有应用数据（搜索索引、设置、窗口布局、同步凭据；自动 ZIP 备份仅通过额外的复选框删除）——你的仓库文件夹在任何情况下都不受影响。

## 界面介绍

- **左侧边栏**——四个视图：**文件**（文件树）、**标签**（仓库中所有的`#标签`）、**书签**和**数据库**（仓库中所有的`.base`，按文件夹分组——点击即可打开）。顶部是大大的**新建**按钮（新建笔记，以及**更多选项**中的新建文件夹、新建数据库、日记）。底部是仓库切换器、**打开日记**和**设置**。四个视图旁边的双箭头按钮可一键折叠或展开所有文件夹；编辑器 ⋮ 菜单中的 **在文件树中显示** 会直接在文件树中定位当前笔记。在**文件**视图中，顶部会显示当前仓库的名称和图标，文件树上方的**最近打开**栏可以一键访问你最近打开过的笔记。
- **标题栏**——你打开的标签页。标签页可以通过拖动重新排序，也可以在编辑器窗格之间移动。
- **编辑器区域**——你在这里阅读和写作。通过标签页菜单（**向右分屏** / **向下分屏**）或快捷键`Ctrl+Alt+V` / `Ctrl+Alt+S`，可以把编辑器分成两个窗格，例如让一篇笔记和一个数据库并排显示。
- **右侧边栏**——四个分区，可拖动调整顺序：**日历**（日记）、**大纲**（当前笔记的标题）、**反向链接**（谁链接到这里）和**属性**（笔记的Frontmatter）。
- **状态栏**——字数/字符数统计、同步状态（本地/在线/离线）和保存状态（**保存中…** / **已保存**）。

## 三种编辑器模式

在编辑器右上方切换模式：

| 模式 | 用途 |
|---|---|
| **阅读模式** | 完全渲染的视图，用于阅读和导航。链接会直接在Plainva内打开。 |
| **实时预览** | 写作时的默认模式：Markdown会随输入即时渲染；格式符号只在你正在编辑的位置才会显示。 |
| **Markdown源码** | 不做渲染的原始文本——用于完全掌控内容。 |

笔记会以哪种模式打开由你决定：在**设置 → 应用 → 编辑器与笔记**中选择**默认视图**（阅读、实时或源码）。在编辑器中手动切换模式，仅对该文件在当前会话内生效。

你也可以在**适读宽度**和**全宽**之间切换。

## 文件树基础

- **新建：** 右键点击文件夹→**在此处新建笔记**、**新建文件夹**或**新建数据库（.base）**。大大的**新建**按钮会在当前选中的文件夹内新建（如果选中的是文件，则在其所在的父文件夹内新建）。
- **选择：** 单击选中，`Ctrl`+单击可单独添加/移除选中项，`Shift`+单击可选择一个区间，鼠标中键点击会在新标签页中打开。
- **右键菜单：** 包括**重命名**（会更新整个仓库中的链接）、**创建副本**、**分屏打开（右侧）** / **分屏打开（下方）**、**添加书签**、**复制路径**、**在文件管理器中显示**、**删除**。
- **多选：** 删除多个项目时只会确认一次，拖动可对整个选中范围进行创建副本和移动操作。被删除的项目会进入操作系统的回收站。
- 新建的笔记会自动以根据文件名生成的`# 标题`开头。
- 文件夹自己的`index.md`（该文件夹的概览）在文件树中会排在该文件夹的**最上方**，位于其子文件夹和文件之上——而不是像其他笔记一样按字母顺序排列。

## 日记

左侧**操作栏**中的**日记**按钮会打开或创建今天的笔记。可以在**设置 → 仓库 → 内容与结构**（点击输入框旁的 **选择文件夹…** 可直接在仓库中选择文件夹）中配置基础文件夹、日期格式和可选的模板。

右侧的**日历**是一个按天呈现的概览：**点击**某个日期会打开[日历标签页](Calendar_and_Tasks.md)并定位到该日；**右键点击**则会打开一个菜单，菜单顶部标明该日期，并提供**打开日历**、**日记**以及当天的日程和到期任务。带有日记的日期上会有一个小小的**太阳图标**，带有日程的日期则按对应日历显示彩色圆点。**今天**按钮会返回到当前月份；点击月份标签会打开一个快速选择月份和年份的界面。你还可以在那里启用**显示周数**，以添加一列ISO周数——该设置会被记住。

## 设置

**设置**（最左侧操作栏底部的齿轮图标，或`Ctrl+,`）可以通过右上角的**X**、`Esc`键或点击窗口外部关闭。更改会立即自动保存——只有云端凭据需要通过**云账户**区域中的**登录**才会生效（参见[设置同步](Sync_Setup.md)）。设置分为两部分；左侧栏中的每个区域都会打开自己的页面，设置以命名的分组卡片形式呈现：

- **应用**——适用于全应用范围的一切设置，分为五个区域。**外观**：**主题**选择器以预览卡片的形式呈现——除了**深青**（默认）之外，还有**Nord**、**Solarized**、**Gruvbox**、**Catppuccin**、**纸张**（类似电子墨水，格外平静）、**深褐**（暖色调纸张）、**森林**、**午夜**（OLED纯黑）、**高对比度**以及**磷光绿**/**磷光琥珀**（带有细微扫描线的复古终端风格）；此外还有**模式**（**浅色**/**深色**/**跟随系统**；像**午夜**这样的单模式主题会固定模式，此时标题栏中的浅色/深色切换按钮会暂时失效）、**语言**、**每周开始于**、**密度**和**界面缩放**。**编辑器与笔记**：**默认视图**、**内容字号**和**内容字体**。**启动与行为**：自动打开上次的仓库、兼容性警告提示。**更新**：Plainva会在启动时静默检查新版本，发现更新时显示提示——点击它即可立即下载并安装更新（该提示会一直显示，直到Plainva重启）。可通过**启动时检查更新**关闭。**关于与诊断**：版本信息、**系统钥匙串**状态、**性能指标**、**导出诊断信息…**（不含笔记内容）和**报告问题**。快捷键始终可以通过`F1`或左下角的**显示快捷键**查看。
- **仓库**——所选仓库在侧栏中显示为一张小卡片（当前仓库带有圆点）；有多个仓库时，卡片下方的**切换**会打开一个选择列表。下方是每个仓库的区域：**云账户**是所有云端登录的唯一入口——**连接账户…**用于选择服务商（Microsoft、Google、Nextcloud、Dropbox、S3、WebDAV或某个邮箱）以及该账户应承担的服务（**文件**、**日历与任务**、**邮件**）。服务区域**同步**（参见[设置同步](Sync_Setup.md)）、**日历**（参见[日历与任务](Calendar_and_Tasks.md)）和**邮件**（参见[邮件捕获](Email_Capture.md)）只有在已连接的账户承担对应服务时才会出现。始终存在的区域：**内容与结构**（**每日笔记**、**模板与任务**包括**模板文件夹**、**OKF（Open Knowledge Format）**——参见[OKF](OKF.md)——以及**扩展数据库**）、**备份与版本历史**和**维护**（**重建索引**、恢复已删除的文件、仓库统计）。

## 自定义界面

- **切换侧边栏**：通过标题栏中的两个按钮，或使用`Ctrl+Alt+B`（左）/`Ctrl+Alt+R`（右）——非常适合专注写作。Plainva会记住该状态。
- **命令面板**：`Ctrl+P`打开**命令**——输入内容并按`回车`执行（新建笔记、日记、分屏、侧边栏、**立即备份**等等）。
- **密度**：在**设置 → 应用 → 外观**中，可以在**舒适**和**紧凑**之间选择——紧凑会压缩列表、菜单和表格行；笔记内容不受影响。
- **内容字体**：在**设置 → 应用 → 编辑器与笔记**中设置**内容字号**（12–24 px）和**内容字体**（**主题默认**、**衬线**、**无衬线**、**等宽**，或通过**自定义…**填入已安装字体的名称）——这只会缩放编辑器和阅读视图，界面本身保持不变。
- **界面缩放**：把整个界面缩放到80%至150%之间——在**设置 → 应用 → 外观**中设置，或使用`Ctrl+Plus`/`Ctrl+Minus`调整（`Ctrl+0`重置）。
- **无原生弹窗的对话框和提示**：确认操作会显示为按你的主题样式渲染的Plainva对话框（破坏性操作带有红色按钮），简短提示则以右下角的轻量吐司通知呈现——不再有系统弹窗。

## 关系图

通过**Ctrl/Cmd+Shift+G**（或右侧边栏中的**关系图**分区），你可以把你的仓库看作一张地图：文件夹是气泡，笔记是节点，关联是带标签的边——还包括清理模式和时间回放。详见[关系图](Graph.md)。

## 记住右侧边栏设置

**大纲**、**反向链接**和**属性**等空的上下文区域会自动折叠，但不会覆盖其全局偏好。整个右侧边栏也会记住一个针对笔记的全局设置；没有笔记上下文的视图只会暂时关闭它。
