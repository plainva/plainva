# 日历与外部任务

更新日期：2026-07-20

Plainva可以连接你已有的日历和任务账户——**CalDAV**（Nextcloud、Fastmail、mailbox.org等）、**Google**（日历+Tasks）和**Microsoft**（Outlook日历+To Do）——并在两个方向上与它们协同工作。你的笔记始终是中心：日程可以变成会议笔记，外部任务列表则会作为普通笔记映射进你的[默认任务数据库](Tasks.md)。

## 连接一个账户

打开**设置 → 仓库 → 云账户 → 连接账户…**，选择一个服务商，并在服务选择步骤中勾选**日历与任务**：

- **Nextcloud / CalDAV**：服务器地址、用户名和一个**应用专用密码**（在Nextcloud中：设置 → 安全 → 设备和会话）。无需注册，也不需要密钥——对于Nextcloud，Plainva会直接从服务器地址推导出CalDAV地址（对于其他CalDAV服务器，请使用**WebDAV / CalDAV**图块，或**高级：分别设置各个端点**）。
- **Apple iCloud、Yahoo、AOL、Zoho、Fastmail、mailbox.org、Yandex、Mail.ru**：专属图块，已预先填好日历地址——只需邮箱地址加一个**应用专用密码**即可，无需服务器字段（对Apple来说应用专用密码是必需的；助手会链接到该服务商的指南）。提示：Yahoo自己也承认其CalDAV服务不太稳定——如果它出现异常，并不是Plainva的问题。
- **Google**：需要你自己的OAuth客户端ID（与Google Drive同步相同的自备凭据（BYO）模式——参见[Drive指南](Google_Drive_BYO_Guide.md)）。此外，请在你的Google Cloud项目中启用*Google Calendar API*和*Google Tasks API*，并在同意屏幕中添加它们的权限范围。浏览器会打开以获取同意；连接操作会在保存任何内容之前先验证账户。
- **Microsoft**：只需点击**使用 Microsoft 登录…**并在浏览器中确认——无需任何设置。一个Microsoft账户还可以在同一次连接中承担**文件**（OneDrive）和**邮件**。

助手会显示每项服务的状态（"已连接——找到 n 个日历"）。你之后可以在**日历**区域管理**日历**的选择（勾选的会出现在日历标签页中）和**任务列表**（默认故意不勾选——勾选一项会启动下方描述的任务同步）；**会议文件夹**（会议笔记的创建位置）和**默认日历**也在那里。密码和令牌保存在你操作系统的钥匙串中。

## 日历标签页

从左侧**操作栏**（日历图标）或**命令面板**（**打开日历**）打开它。顶部的切换控件提供五种视图：**日**、**3天**和**周**视图都会显示一个**时间网格**，左侧带有小时刻度；日程以色块形式出现在其开始时间处，色块高度对应持续时长，重叠的日程会并排显示，一条红线标记"现在"。全天日程以及（开启任务叠加后）到期的任务会显示在网格上方的条带中。**月**视图会显示月历网格（每个日历对应一个彩色圆点），并在右侧显示所选日期的单日时间网格。**议程**视图按天分组列出接下来几周的安排。**今天**可跳回当前；箭头按当前所选周期（一天、三天、一周或一个月）翻页。一周的第一天遵循**每周开始于**设置（设置 → 应用 → 外观：周一、周六或周日）——该设置同样适用于侧边栏日历。视图每隔几分钟会自动刷新；点击刷新按钮可以强制刷新。已结束的日程会显示得更**暗淡**（如同Google日历），让今天剩余的安排更加醒目。

- **新建日程**：**点击时间网格中的空白位置**会打开一个小巧的快速新建弹窗（标题、时间、日历、地点）——**保存**会立即创建，**更多选项**会打开完整的日程对话框。在网格上**拖动**可以设置持续时长。顶部的**+**会打开完整的对话框：标题、日历、日期/时间或一个全天区间、地点、说明、颜色，以及一个可选的Outlook风格**重复**。这个颜色只对该日程本身生效，会覆盖它所属日历的颜色（对Microsoft账户没有影响——Outlook不支持逐日程设置颜色）。
- **参与者**：输入一个电子邮件地址并按**回车**（或逗号）将其添加为**标签**；点击×可以移除。重复选项就在日期/时间旁边设置——可以选择频率、间隔、星期几（每周重复时可选），以及结束方式（永不 / 于某天 / 重复 N 次后）；你也可以为已有日程添加或更改重复规则。
- **编辑 / 删除**：在时间网格中**点击某个日程**会打开对话框，预填其现有信息，并带有**会议笔记**和**删除**操作。更改会经过一项安全检查后被写入服务商：如果该日程在此期间已在远端发生变化，Plainva会刷新，而不是覆盖。
- **移动 / 调整大小**：你可以直接在时间网格中**拖动**日程——拖动日程主体会重新安排其时间（在周/3天视图中也可以拖到另一天），拖动它的**下边缘**则会更改持续时长。新的时间会立即写入服务商（重复日程目前仍只能通过对话框进行编辑）。
- **RSVP与回复**：当你受邀参加某个日程时，对话框会让你**接受**、标记为**暂定**或**拒绝**——Plainva会通过服务商（Google/Microsoft/CalDAV）发送你的回复。**参会人列表**会显示谁已接受或拒绝（这是回复信息的接收端）。
- **邮件邀请**：当日程带有参与者时，勾选**通过电子邮件通知参与者**。在Google上，Plainva会请求Google发送其原生邀请（仍是同一个日程，因此对方的回复会同步回你的日程）；Microsoft会自动通知参与者。对于CalDAV——或者想从自己的邮箱发送一份副本——日历的**通过邮件发送**操作会打开邮件撰写窗口，并附上一份符合标准的iCalendar邀请文件，这样Gmail和其他邮件客户端就会把它显示为带有是/也许/否选项的日程。
- **在其他日历中占用**：日程上的**复制**图标操作（或其对话框中的**在其他日历中占用**按钮）会把它镜像到你的一个或多个其他可写日历中——可以作为不透明的**忙碌**占位符，也可以**含详情**（Notion Calendar风格）。重复日程会连同其重复规则一起镜像，因此占用条目也会跟着重复。
- **重复日程**带有一个重复徽章。编辑或删除某一次实例时，Plainva会询问**"仅此日程"**（创建一个例外，即跳过这一次）还是**"所有日程"**（更改整个系列）。Plainva绝不会重写已有的重复规则。
- **显示任务**（刷新按钮旁，仅当设置了默认任务数据库时出现）：把你[默认任务数据库](Tasks.md)中带截止日期的条目叠加显示在时间网格的条带和月历网格上；已完成的任务会带删除线。默认关闭，你的选择会按设备记住。

## 日程 → 会议笔记

任意日程上的笔记图标会创建（或重新打开）它的**会议笔记**——一篇存放在你会议文件夹中的普通笔记，命名为`YYYY-MM-DD 标题.md`，预先填入日期、地点和参会人，外加Frontmatter中一个把它和该日程关联起来的小`plainva.pim`标记。再次点击同一个日程，总是会打开同一篇笔记；一篇恰好同名的自建笔记永远不会被碰到。

## 你的任务数据库中的外部任务列表

在一个已连接的账户上勾选一个**任务列表**，它的任务就会作为笔记出现在你的[默认任务数据库](Tasks.md)中：标题会变成笔记（H1），截止日期会落入数据库的日期列，完成状态会映射到数据库的**已完成复选框属性**（状态列会随之跟随；没有复选框列的数据库会使用状态约定——第一个选项=未完成，最后一个选项=已完成）。这个同步是双向的，并且按字段进行：

- 编辑笔记（标题、截止日期、状态）→ 更改会被推送到服务商。
- 远端更改了任务 → 笔记会随之更新。
- 如果两侧都有更改，该字段以你的本地更改为准；其余部分则以远端为准。

两条安全规则保护着你的数据：**删除笔记绝不会删除远端的任务**（它只是不再同步，也不会被再次导入），并且**远端删除的任务绝不会删除你的笔记**（它只会变成一篇普通笔记）。重命名或移动一篇任务笔记不会有问题——Frontmatter标记会保持这个关联。

当前的限制：以普通笔记方式创建的任务不会被推送到服务商（请在远端或通过任务数据库创建它们），本页上的一切目前都以桌面版为先。
