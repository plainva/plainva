# 设置同步

更新日期：2026-07-22

Plainva可以选择性地把每个仓库与你自己选择的存储同步——直接从应用内完成，中间没有任何由Plainva运营的服务：你的数据只会在你的电脑和你自己的账户/服务器之间传输。本页逐步说明各服务商的配置方法。

哪些服务大体上可用（包括通过WebDAV或服务商的桌面客户端）在[同步兼容性](Sync_Compatibility.md)中有详细说明。

## 基础知识

- 配置入口在**设置 → 仓库 → 云账户**：**连接账户…**会打开助手——先选择**服务商**，再勾选**服务**（用于文件同步的是**文件**），最后登录。图块概览会按实际普及程度列出各服务商；通过**搜索服务商…**还能找到以预设形式提供的邮件服务商。每个仓库始终恰好由**一个**账户承担**文件**服务。**同步**区域随后会显示已连接的账户及其**云端文件夹**，并管理相关行为（**同步间隔**、队列）；**管理账户**会返回云账户。
- 对于**文件**服务，除了**Microsoft**（OneDrive）、**Google**（Drive）、**Dropbox**、**Nextcloud**、**对象存储 (S3)**以及通用的**WebDAV / CalDAV**之外，图块中还包含**Fastmail**、**mailbox.org**、**Yandex**、**Mail.ru**、**Koofr**和**pCloud**：在这些服务中，只需你的邮箱地址加一个**应用专用密码**即可——服务器地址已经预填好（基于WebDAV；可通过**高级：分别设置各个端点**更改）。
- **从欢迎界面设置已有的在线仓库**：**打开仓库** → **在线仓库**会为每个服务商提供相同的三个步骤——**1. 连接**（登录或输入凭据）、**2. 选择云端文件夹**（也可以在此处通过**新建文件夹**新建一个）、**3. 选择或创建本地文件夹**。你也可以随时在设置中为已经打开的仓库配置同步。
- **在云端新建仓库**：**新建仓库** → **在在线服务中**——先选择起始结构（空白或PARA等模板），然后连接并选择云端的目标文件夹，或通过**新建文件夹**新建一个，最后选择本地文件夹。结构会创建在本地文件夹中，并在首次同步时自动上传。
- 本地保存会立即上传；Plainva会按配置的**同步间隔（秒）**检查远端更改。
- 离线时的更改会排入队列，等到下次联通时传输；状态栏会显示**在线**/**离线**，同步指示器会显示当前状态（点击可**立即同步**）。在长时间同步或首次同步期间，状态栏会以计数形式显示进度（例如**同步 123/540**），让你能看到它正在逐步处理整个仓库。
- 首次连接一个在线仓库时，会有一条一次性提示告诉你：根据仓库大小，首次同步可能需要一些时间——在此期间你可以继续工作。
- 如果两端都修改了同一个文件，Plainva会自动合并（三方合并）。如果无法合并，你的版本会被安全地保存为一个`.CONFLICT`文件——任何内容都不会丢失（参见[常见问题与故障排查](FAQ.md)）。
- **解决冲突**：受影响笔记中的一条横幅（以及文件树中`.CONFLICT`文件右键菜单里的**解决冲突…**）会打开对比对话框——左侧是文件的当前状态，右侧是你被保留下来的版本，可编辑，并支持按块单独采用。**保存右侧版本并解决**会把结果写入文件并清理冲突副本；**保留另一端**则会丢弃你的副本（仍会保留一个版本快照）。同步错误对话框也会列出现有的冲突副本，点击一下即可进入同一个对比界面。
- **批量删除保护**：如果异常多的已同步文件即将在云端被一次性删除（例如因为本地仓库文件夹被清空或移动），Plainva会先暂停这些删除并询问：**在云端删除**会执行删除，**不删除（恢复）**会放弃删除，并在下次同步时从云端恢复这些文件。你在 Plainva 中自行确认的删除不会被拦截——对于大量删除（超过 10 个文件或超过仓库的 20%），Plainva 会在删除前再次向你确认。
- 附件（图片等）也会一并同步。
- **空文件夹**也会同步：在Plainva中新建的文件夹会立刻出现在云端，云端已有的空文件夹最晚会在你其他设备的下一次完整同步中出现。
- 凭据和令牌保存在操作系统的钥匙串中（状态查看：**设置 → 应用 → 关于与诊断 → 系统钥匙串**），绝不会保存在仓库内的文件中。
- **断开连接**会停止该仓库的同步；此操作不会删除任何地方的任何文件。

## WebDAV / Nextcloud

对于自建服务器和大多数云存储来说，这是最简单的方式：

1. 在**云账户** → **连接账户…**中选择**Nextcloud**（或**WebDAV / CalDAV**）图块。
2. 输入**服务器地址**、**用户名**和**密码或应用令牌**——只要可能，就使用应用密码而不是主密码（在Nextcloud中：设置 → 安全 → 应用密码）。
3. **连接**会验证凭据；之后通过**选择文件夹…**选择**云端文件夹**。

**Nextcloud**的特殊之处：一个表单即可同时覆盖文件**和**日历——Plainva会直接从服务器地址推导出WebDAV和CalDAV端点（推导出的地址会显示在助手中；**高级：分别设置各个端点**可以设置独立的URL）。同时勾选两项服务，一次连接即可同时完成两者。

常见的服务器地址（Nextcloud、Koofr、MagentaCLOUD、Storage Box等等）列在[同步兼容性](Sync_Compatibility.md)中。

## Google Drive

Google Drive目前采用自备凭据（"Bring Your Own"）的方式运行：你只需一次性创建一个只属于你自己的免费Google Cloud项目。分步指南见：[Google Drive（BYO）](Google_Drive_BYO_Guide.md)。

简要版本：在**云账户** → **连接账户…**中选择**Google**图块，勾选**文件**服务，输入你Google项目中的**客户端ID**和**客户端密钥**，然后点击**使用 Google 登录…**——登录界面会在浏览器中打开。连接成功后，你可以通过**选择文件夹…**直接从你的Drive中选择**云端文件夹**（包括子文件夹，默认为"Plainva"）。注意：只要Google项目处于测试模式，登录就会在7天后过期，需要在账户详情中通过**重新登录**重新登录。

## OneDrive

Plainva自带了应用注册——你**不再需要自己创建ID**：

1. 在**云账户** → **连接账户…**中选择**Microsoft**图块，并勾选**文件**服务（OneDrive）——如果需要，可以同时勾选**日历与任务**和**邮件**（一个Microsoft账户可以同时承担全部三项服务）。
2. 点击**使用 Microsoft 登录…**，并在浏览器中确认登录。完成——Plainva会创建该文件夹（默认为"Plainva"），并同步其全部内容，包括外部添加的文件。
3. 可选：连接成功后，你可以通过**选择文件夹…**直接从你的OneDrive中选择**云端文件夹**（包括子文件夹）。

可选：通过**使用你自己的应用 ID**，你可以改为填入一个自行注册的客户端ID（例如遇到公司限制时）。详细指南：[OneDrive与Dropbox（BYO）](OneDrive_and_Dropbox_BYO_Guide.md)。

## Dropbox

Plainva自带了Dropbox应用——**无需自己的应用**：

1. 在**云账户** → **连接账户…**中选择**Dropbox**图块（它只承担**文件**服务）。
2. 点击**使用 Dropbox 登录…**，并在浏览器中确认。完成（默认文件夹为`/Plainva`）。
3. 可选：连接成功后，你可以通过**选择文件夹…**直接从你的Dropbox中选择**云端文件夹**（包括子文件夹）。

可选：通过**使用你自己的应用 ID**，你可以改为填入一个自行注册的App key。详细指南：[OneDrive与Dropbox（BYO）](OneDrive_and_Dropbox_BYO_Guide.md)。

## S3兼容存储

适用于AWS S3、Cloudflare R2、Backblaze B2、MinIO、Wasabi、Hetzner等等——纯密钥方式，完全不需要浏览器登录。在**云账户** → **连接账户…**中选择**对象存储 (S3)**图块并填写以下字段：

| 字段 | 含义 |
|---|---|
| **Endpoint** | S3 API的基础URL，例如`https://s3.eu-central-1.amazonaws.com`、`https://<account>.r2.cloudflarestorage.com`，或本地MinIO的`http://127.0.0.1:9000` |
| **存储桶** | 存储桶名称 |
| **区域** | SigV4区域；大多数非AWS存储使用`us-east-1`即可，Cloudflare R2使用`auto` |
| **Access Key ID** / **Secret Access Key** | 来自服务商的一对API密钥 |
| **Key前缀（可选）** | 仓库在存储桶内所处的子文件夹；留空=存储桶根目录 |
| **路径样式URL** | 推荐启用（适用于MinIO、R2和大多数兼容存储）；仅在virtual-hosted风格的AWS存储桶上关闭 |

**Key前缀**（即云端文件夹）可以在连接后通过**选择文件夹…**直接从存储桶中选择。

**连接**后，同步会立即开始。

## 同步加密（密码短语）

> **已在P3替换：** 以下说明不再用于仓库内容。请使用[安全与共享](Security_and_Sharing.md)。此处保留的口令仅保护可选设置与机密信息。

Plainva 可以对从你的设备发往同步服务器的内容进行加密，而你的本地仓库始终保持为Obsidian可以读取的纯Markdown格式。

打开**设置 → 同步 → 同步密码短语与加密**：

1. **设置密码短语。** 这会为仓库创建一个加密密钥，并显示一次性的**恢复代码**——请妥善保存；如果你忘记了密码短语，这是唯一能找回访问权限的方法。从此以后，该仓库同步的**设置**会以加密形式传输。
2. **加密仓库内容**（可选）。**加密**按钮会把每一篇笔记以密文形式重新上传到同步服务器。你的本地文件仍然是纯Markdown，所以本地仓库永远不会有风险——建议先在一个可丢弃的仓库上试一试。上传完成后，使用**完成迁移**，此后就只接受密文。
3. **在另一台设备上**，打开同一个已同步的仓库。Plainva会检测到该仓库已加密，并提示你输入密码短语（或恢复代码）。解锁后，笔记会被解密并出现在本地。

已解锁的密钥会在每台设备上缓存。开启**每次启动都要求输入密码短语**，可以改为在每次重启后重新输入；使用**锁定**可以从此设备上移除缓存的密钥。

**同步设置**会传输共享的仓库设置和账户元数据；本地路径、布局和运行时数据仍保留在各自设备上。**同步账户密钥**是单独的可选项，仅用于允许的静态应用密码和 BYO 凭据；OAuth 令牌绝不会共享。加密状态会依次显示**准备**、**迁移**、**严格**、**解密**和**密钥轮换**。移动设备也可以用同一密码短语解锁加密仓库。

## 错误和自动重试

同步错误窗口会保留失败尝试的确切信息，即使自动重试已经改变了实时状态。它会显示重试正在进行或已经成功。只有身份验证错误才会建议重新连接；网络、超时和提供商错误会保留具体原因并自动重试。
