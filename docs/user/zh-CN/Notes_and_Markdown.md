# 笔记与Markdown

更新日期：2026-07-14

Plainva中的每一篇笔记都是一个普通的Markdown文件（`.md`）。本页说明如何舒适地写作，以及文件里实际保存了什么——这正是让你的笔记具有可移植性的原因：任何文本编辑器、Obsidian，甚至git diff都能读懂它们。

## 核心原则：一切皆文本

无论你在Plainva中看到的是什么——格式化文本、表格、属性、图标——它们都以开放的文本形式存储：

```markdown
---
type: Note
okf_version: "0.1"
tags: [project]
plainva:
  icon: "🚀"
  header_color: "#2f6f6f"
---
# My Project

A **bold** thought with a link to [[Another Note]].

- [ ] First task
```

`---`之间的部分是**Frontmatter**（YAML）：笔记的属性就保存在这里。下面是普通的Markdown正文。Plainva特有的呈现方式（图标、页眉颜色）统一归在`plainva:`这一个键下——其他程序会直接忽略它。

## 在实时预览中写作

**实时预览**是默认模式：Markdown会随输入即时渲染，同时始终保持可编辑。

### 斜杠菜单

在行首输入`/`即可打开插入菜单，它按分区分组：

- **基础块**——文本、一级标题至六级标题、无序列表、有序列表、待办清单、引用、代码块、表格、分割线、**公式 (LaTeX)**、**Mermaid 图表**
- **格式**——粗体、斜体、删除线、行内代码、高亮、**表情符号**
- **链接与媒体**——链接、内部链接、图片（网络）、内部图片、嵌入、嵌入数据库、创建内联数据库
- **文档**——文档图标、页眉颜色、插入模板
- **标注**——13种变体（笔记、信息、待办、摘要、提示、成功、问题、警告、失败、危险、Bug、示例、引用）

### 更多写作辅助功能

- **选中工具栏**——选中一段文字后，会出现一个小工具栏，提供**粗体**、**斜体**、**删除线**、**行内代码**、**高亮**和**链接**。
- **`@`提及**——在文中任意位置输入`@`，即可插入**日期**（今天、明天、昨天，或**选择日期…**，以ISO日期格式存储）、指向某条**笔记**的链接，或嵌入一个**数据库**。
- **表情符号**——**表情符号**斜杠命令（`/emoji`）会在光标处打开表情符号选择器；也可以输入`:name`（例如`:rocket`）获得内联建议。无论哪种方式，Plainva插入的都是真正的表情符号**字符**（可移植的Unicode），而不是`:shortcode:`——这样笔记在Obsidian、GitHub等任何地方都能保持可读。（这与保存在Frontmatter中的笔记**文档图标**是两回事。）
- **块柄**——鼠标悬停在每个段落时，左侧会出现一个块柄：拖动它可以移动该块，点击它可以打开**块操作**（**转换为**文本/标题/列表/待办/引用/代码块、**创建副本**、**上移**/**下移**、**删除块**）。如果你把一个列表拖到另一个同类型列表旁边，Plainva会插入一条不可见的分隔行`<!-- -->`，让两个列表保持独立——在Markdown中，即使有空行，同样式的列表也会合并（在Obsidian中同样如此）。
- **表格**——以带点击编辑单元格的组件形式渲染。单元格显示会渲染格式（**粗体**、*斜体*、`代码`、高亮）、可点击的链接（`[[内部链接]]`、网址）以及作为换行符的`<br>`；在编辑时你看到的是原始文本。表格菜单提供插入/删除行和列，以及对齐方式（**左对齐**/**居中对齐**/**右对齐**）。
- **列表可自动延续**（按Enter键会插入下一个列表标记），代码块具有语言感知的语法高亮（在阅读模式下同样如此），粘贴的内容会被转换为Markdown（智能粘贴），标题可以折叠。
- 当前笔记内的**查找和替换**：`Ctrl+F`（参见[搜索](Search.md)）。

## 链接与反向链接

- **内部链接**：`[[笔记名称]]`（内部链接）——可通过斜杠菜单或`@`加内置笔记搜索插入。经典的Markdown链接`[文本](path.md)`同样有效。
- **反向链接**：右侧边栏的**反向链接**分区显示哪些笔记链接到当前笔记——按来源文件分组，多次出现时会显示计数。
- **重命名时保护链接**：当你在文件树中重命名一个文件时，Plainva会更新整个仓库中指向它的每一个链接（像`#章节`这样的锚点会被保留），并报告："已更新M个文件中的N个链接为新名称。"

## 属性（Frontmatter）

右侧边栏的**属性**分区把笔记的Frontmatter显示为一个表单。**添加属性**可以新建属性；每个属性都有一个**字段类型**：

| 分组 | 类型 |
|---|---|
| **基础** | 文本、数字、复选框、日期、日期和时间 |
| **选择** | 单选、状态、多选 |
| **列表与关联** | 列表、标签、关联 |
| **网络与联系方式** | URL、邮箱、电话 |

选择类型可以携带固定选项，每个选项有一个**颜色**，**状态**类型还可以设置**分组**/阶段——这些选项列表在数据库（`.base`）中管理，参见[数据库（.base）](Databases_Base.md)。

有两个字段受到保护：`type`和`okf_version`是由Plainva管理的**OKF系统字段**——`type`的值可以从已知类型的下拉列表中选择，而名称/字段类型/删除都被锁定（背景说明见[OKF](OKF.md)）。

## 文档图标与页眉颜色

每篇笔记都可以携带一个图标（Notion风格，位于标题上方，标签页和文件树中也可见）和一条全宽的颜色条：

- 在实时预览中，把鼠标悬停在标题上方即可看到：**添加图标** / **添加页眉颜色**（之后会变为：**更改图标** / **更改页眉颜色**）——也可以使用斜杠命令**文档图标**和**页眉颜色**。
- 图标选择器有两种模式：**Emoji**和**图标**（Lucide图标集，可选择颜色）。
- 两者都保存在Frontmatter的`plainva:`键下（`icon`、`icon_color`、`header_color`）——纯粹的呈现信息，不会影响其他程序。

## 模板

在**设置 → 仓库 → 内容与结构**（点击输入框旁的 **选择文件夹…** 可直接在仓库中选择文件夹）中设置**模板文件夹**。之后可以通过`Ctrl+Alt+T`或斜杠命令**插入模板**来插入模板。模板完全定义新文件的内容——包括Frontmatter：如果模板自带`type`，模板的值优先生效。插入到现有笔记时，模板的 frontmatter 会被省略——只插入内容。

**占位符**：模板会插值`{{title}}`（笔记的标题）、`{{date}}`和`{{time}}`。当你*插入*一个模板时，还会额外解析两个占位符：`{{cursor}}`标记之后光标的落点，`{{prompt:Label}}`会向你询问一个值（显示为*Label*）并插入你的回答。从模板*新建*笔记时，`{{cursor}}`会被移除，`{{prompt:…}}`则留空。

新建模板随处可用：命令面板（`Ctrl+P`）提供**创建新模板**（会打开一个全新的模板供编辑）和**将当前笔记保存为模板**（把当前打开的笔记复制到模板文件夹）。模板就是普通的Markdown文件——可以直接在文件树中编辑、重命名或删除它们。

## 日记

**打开日记**（侧边栏）或点击**日历**中的某一天，会使用你设置的日期格式，在已配置的日记文件夹中创建今天的笔记，也可以选择性地基于模板创建。

## 任务、公式、图表与脚注

- **任务复选框**：`- [ ] 任务`在任何地方都会渲染为一个复选框——而且在**阅读模式**下可以直接点击它：Plainva会把`[x]`或`[ ]`写回文件。
- **数学公式（LaTeX）**：行内的`$E = mc^2$`和作为代码块的`$$…$$`在阅读模式**和**实时预览中都会渲染为公式（KaTeX）。当光标停留在公式内部时，会显示其语法；点击一个已渲染的公式即可打开它进行编辑。只有源码模式会始终显示原始语法。你不必死记`$$…$$`代码块——斜杠命令**公式 (LaTeX)**（`/katex`）会插入它，并把光标放在其中。
- **Mermaid图表**：语言标记为`mermaid`的代码块（最快的方式是使用斜杠命令**Mermaid 图表**，`/mermaid`）会在阅读模式和实时预览中被绘制为图表——点击图表即可显示代码进行编辑：

  ````markdown
  ```mermaid
  graph TD
    Idea --> Note --> Knowledge
  ```
  ````

- **脚注**：`文本[^1]`加上文末的`[^1]：脚注内容。`——阅读模式会渲染出引用标记和带跳转标记的脚注列表。最快的方式是使用**脚注**斜杠命令（`/footnote`）：它会插入下一个空闲的引用标记，并直接跳转到笔记末尾的定义处。

## 打印与另存为PDF

编辑器的**⋮**菜单和命令面板（`Ctrl+P`）中都有**打印 / 另存为PDF…**：打印始终使用阅读视图（如果当前处于实时预览/源码模式，Plainva会先自动切换过去）。在系统对话框中，你可以选择"另存为PDF"而不是打印机。

## 导出笔记

- **导出为 Markdown…**（编辑器**⋮**菜单或命令面板）：通过系统对话框把笔记的副本保存到任意位置——例如交给另一个程序使用。链接的附件（图片）不会随之导出；如果笔记引用了附件，Plainva会显示一条简短提示。
- **PDF**：使用上方的**打印 / 另存为 PDF…**，并在系统对话框中选择"另存为PDF"。

## 在其他编辑器中打开笔记

你的笔记是普通的`.md`文件，因此任何Markdown编辑器都能打开它们。编辑器的**⋮**菜单中有**在默认应用中打开**，会把当前笔记交给你系统默认用于打开Markdown文件的应用程序（Byword、MacDown、VS Code等）。Plainva会持续监视该文件，因此你在那里所做的修改会自动出现在这里。

## 图片与附件

- **插入**：斜杠命令**内部图片**（在仓库中搜索并嵌入）或**图片（网络）**（通过URL插入）。也可以直接从剪贴板**粘贴**一张图片（Ctrl+V）——它会被保存在笔记旁边并嵌入。你还可以**把文件从文件资源管理器拖拽到编辑器中**：图片会被嵌入（`![[…]]`），其他文件会被复制进来并链接（`[[…]]`）。
- **查看**：图片文件（PNG、JPG、GIF、WebP、SVG、BMP、AVIF）会在内置的图片查看器中打开，支持**放大**/**缩小**、**适应窗口**和**实际大小（1:1）**。
- **编辑**：**编辑**按钮会打开图片编辑器，提供**裁剪**、旋转/翻转、**调整大小**、绘图工具（**画笔**、**箭头**、**矩形**、**文字**）以及**撤销**/**重做**。可以直接保存，也可以**另存为副本…**。可编辑的格式是PNG、JPG和WebP；其他格式仅支持查看。
- 其他附件双击后会在系统的默认程序中打开。

## 那Obsidian呢？

一切仍然是带有标准Frontmatter的标准Markdown。Obsidian可以完整地打开这些文件；它会在属性面板中把打包的`plainva:`键显示为一个不可编辑的对象——这是刻意如此，且无害。
