# 常见问题与故障排查

更新日期：2026-07-11

对最常见问题的解答——从Obsidian兼容性到冲突文件和备份。

## 基础问题

### 我的数据保存在哪里？

完全只在你自己手中：仓库就是你电脑上一个普通的Markdown文件文件夹。Plainva不运行任何自己的服务器，也不会在任何地方保留副本。如果你启用了同步，数据只会直接在你的电脑和*你自己的*存储（你的Nextcloud、你的OneDrive、你的存储桶……）之间传输。凭据保存在操作系统的钥匙串中。

### 我可以同时使用Plainva和Obsidian吗？

可以——这是一项核心承诺，但也有一点需要坦诚说明。Plainva写入的是带有标准Frontmatter的普通Markdown；所有Plainva特有的内容都统一归在`plainva:`键下（在笔记和`.base`文件中），Obsidian在打开文件时会直接忽略它。Obsidian会把`plainva`键显示为属性中一个不可编辑的对象——这是无害的。像看板或日历这样的Plainva专属视图，在Obsidian中会显示为普通表格。

限制在于：**打开始终是安全的，编辑则不一定。** 已有的Obsidian仓库可以在Plainva中放心地打开和编辑——不会有任何内容被迁移或重新格式化。但一旦某个仓库使用了Plainva的功能（例如看板、关联或反向关联列这类数据库扩展功能，以及由Plainva管理的`index.md`文件），在Obsidian中编辑这些特定文件就可能破坏Plainva的相应功能，因为Obsidian并不认识`plainva:`扩展。未使用Plainva扩展功能的笔记，随时随地都可以编辑。第一次使用这类扩展功能时，会有一个提示对话框（**Plainva扩展功能**）说明这一点；可以在**设置 → 应用 → 启动与行为**中关闭它。

### Plainva会修改我已有的仓库吗？

不会擅自修改。已有文件只有在你明确启动某项操作时才会被改动（例如[OKF转换](OKF.md)——带有预览和备份）。只有新建的文件才会自动获得那一小段OKF Frontmatter头。

## 文件与编辑

### 我删除了某些内容——它彻底消失了吗？

不会，而且是双重保障：每次删除前，Plainva都会先把文件保存为一个快照——右键点击仓库名称 → **恢复已删除的文件…**即可在应用内把它找回来。此外，被删除的文件和文件夹还会进入操作系统的回收站（对于整个文件夹，回收站是主要的恢复途径）。详情见：[Backups与版本历史](Backups_and_Versioning.md)。

### 我的笔记有更早的版本吗？

有：Plainva会在你编辑时自动创建文件版本。右键点击一个文件 → **版本历史…**会显示所有快照，并提供对比视图和**恢复**功能。此外，Plainva每天还会把整个仓库备份为一个ZIP，保存在仓库文件夹之外。详情见：[Backups与版本历史](Backups_and_Versioning.md)。

### 为什么我的index.md是只读的？

它是由Plainva生成的，并会自动保持更新（可通过横幅"此index.md由Plainva管理…"识别）。**仍要编辑**会把它永久交由你手动维护——此后它不会再自动更新。详情见：[OKF](OKF.md)。

### 在数据库中重命名一个属性会发生什么？

新名称会被写入**每一篇匹配笔记**的Frontmatter中（需要确认，并会显示进度）。删除属性也遵循同样的原则：勾选**同时从笔记的Frontmatter中移除**会一并清理源笔记。两者都会作用于你的文件——这正是它们的用途所在。

### 我可以撤销OKF转换吗？

在做任何更改之前，向导都会把文件备份到`.plainva/backups/okf-conversion-<timestamp>/`。最终报告会给出确切的文件夹名称，你可以从那里把单个文件复制回来。转换前也可以先使用**预览（不做更改）**。

## 同步

### 什么是.CONFLICT文件？

如果同一个文件在这里和另一台设备上同时被修改，Plainva会先尝试自动合并两个版本。如果无法合并，**你的**版本会被安全地保存为原文件旁边的一个`.CONFLICT`文件——任何内容都不会丢失。冲突文件会在文件树中标记出来；右键点击可以选择**保留此版本**（冲突版本会替换原文件）或**丢弃冲突**。

### 我的Google登录总是过期

在"自备凭据"的配置下，你的Google项目会保持在测试模式，Google会在7天后结束该会话。Plainva会在后台自动续期令牌，但一旦过期，请在同步设置中使用**重新连接**。详情见：[Google Drive（BYO）](Google_Drive_BYO_Guide.md)。

### 我的仓库位于OneDrive/Dropbox/iCloud文件夹中，Plainva的表现有些奇怪

请在服务商的同步客户端中，把仓库文件夹设置为"始终保留在此设备上" / "可离线使用"。仅在线的占位文件（按需下载、"online-only"）会干扰索引和同步。详情见：[同步兼容性](Sync_Compatibility.md)。

### 我处于离线状态——我的更改会怎样？

它们会照常本地保存，并收集到一个队列中；一旦连接恢复，Plainva会自动传输它们。状态栏会显示**在线**/**离线**。

### 状态栏显示离线，但我明明有网络

那么很可能是同步连接本身出了问题——通常是因为登录已过期，或者凭据发生了变化（例如Google Drive）。点击状态栏中的**离线**，或点击仓库名称旁边的警告三角形：对话框会显示确切的错误信息，**打开同步设置**会直接带你进入对应服务商的表单，你可以在那里重新连接（例如**重新连接**）。每次点击也会立即触发一次同步重试。

## 应用

### 为什么 F5 不刷新，浏览器的右键菜单在哪里？

Plainva 是桌面应用，而不是网页。刷新快捷键（F5、Ctrl+R）被有意禁用——刷新会丢弃你打开的标签页和未保存的修改。WebView 内置的右键菜单也被隐藏；在选中的文本上右击仍会提供**复制**，而文件树、标签页和表格仍保留各自的右键菜单。

### 为什么我看不到任何动画？

Plainva会遵循你系统的"减弱动态效果"设置。如果过渡和特效缺失（按钮、菜单和高亮都不会有动画），说明你的操作系统中关闭了动画效果。在**Windows**上：设置 → 辅助功能 → 视觉效果 → 打开**动画效果**。在**macOS**上：系统设置 → 辅助功能 → 显示器 → 关闭**减弱动态效果**。

### 如何更改语言？

**设置 → 应用 → 外观 → 语言**（目前支持德语和英语）。

### "检查更新"什么都没找到

只要目前还没有公开发行版，更新检查就会提示："目前还没有公开的更新（发行版）。"这不是错误。

### 有什么隐藏功能吗？

星际舰队对传闻概不置评。但据说标题栏里的那个标志，会对持续不断的敲击有所回应——而知道正确暗号的人，之后会以一种全新的方式看待Plainva。有人说：一共有四个。
