# 邮件捕获

更新日期：2026-07-20

Plainva可以读取你的邮箱，把邮件中的知识提取到你的仓库中——从0.4.0版本开始，也可以撰写和发送邮件。重点仍然是把邮件**捕获**为笔记；通过**IMAP**连接的邮箱在你配置发送功能之前，始终只用于捕获的只读访问（邮箱中的任何内容都不会改变，甚至连未读标记也不会）。

## 连接一个邮箱

**设置 → 仓库 → 云账户 → 连接账户…**，然后选择服务商：

- **Microsoft** — 适用于Outlook.com和Microsoft 365：在服务选择步骤中勾选**邮件**（如果需要，可以同时勾选**文件**和**日历与任务**——一个账户、一次登录），然后直接在浏览器中登录，完全无需应用专用密码，也无需IMAP。Plainva为此使用统一的Plainva应用注册（你可以在账户详情中选择性地填入自己的应用ID）。读取邮箱、捕获以及**直接发送**都通过Microsoft登录完成。
- **Apple iCloud**、**Yahoo**、**AOL**、**Zoho**、**Fastmail**、**mailbox.org**、**Yandex**、**Mail.ru**——专属图块：邮箱地址加一个**应用专用密码**，服务器已预先填好（大多数这些图块也允许在同一步骤中勾选**日历与任务**——一个应用专用密码即可用于所有已选服务）。助手会分别链接到各服务商创建应用专用密码的官方指南。
- **邮件服务器 (IMAP)** — 适用于所有其他服务商：主机、端口和一个密码或**应用专用密码**。现成的预设覆盖了来自世界各地的服务商——从**web.de**/**GMX**和**T-Online**，到**Orange**、**Libero**、**WP**、**Seznam**和**Comcast**，再到**QQ Mail**、**NetEase**、**Naver**和**Yahoo! JAPAN**；为此**服务商**下拉框带有搜索行，输入你的地址会自动选中匹配的预设。如果某个服务商有特殊之处，助手会在表单正下方说明：有些服务商要求使用**应用专用密码**或**授权码**代替账户密码，另一些则需要先在服务商的设置中启用IMAP——每一项都附有官方指南的链接。对于Gmail来说，那就是`imap.gmail.com`，端口`993`，配合一个来自[myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)的应用专用密码（需要开启两步验证）——无需OAuth，也无需验证；助手本身会针对Gmail地址提示这一点。**Outlook.com邮箱**已不能再通过密码方式的IMAP连接（Microsoft已关闭该途径）——该预设会指向**Microsoft**图块。**Proton Mail**只能通过本地运行的付费Proton Mail Bridge使用（有专属预设）。若要直接发送，可以填写一个SMTP主机。

连接操作会在保存任何内容之前先验证登录信息；凭据会存入你操作系统的钥匙串。已连接的邮箱和捕获设置之后会在**邮件**区域中找到：**邮件文件夹**设置决定了已捕获的邮件会存放在哪里（默认为`Mail`）。

## 阅读邮件

从左侧**操作栏**（信封图标）或**命令面板**（**打开邮件**）打开邮件标签页。列表会按最新优先显示你的收件箱（未读邮件加粗，**加载更多**会翻页加载更多）。选中一封邮件，会在**沙盒查看器**中打开它：

- **外部内容会被拦截**——跟踪像素、外部图片和样式加载器会被移除并计数（"已拦截外部内容（n）"）。只有自包含的内联图片才会显示。计数旁的**显示图片**会临时显示这封邮件的https图片一次；邮件设置中的**始终加载外部图片**则会把这一行为变成长期启用的选项。请注意：加载外部图片会让发件人看到你的IP地址以及你打开邮件的时间——这也是默认拦截外部内容的原因。
- 链接以纯文本形式显示，在查看器中不可点击。
- 脚本和表单永远不会运行。邮件会在一个具有严格内容策略的隔离框架中渲染。

附件会以名称和大小列出；下方的原始`.eml`文件（见下文）则完整包含它们。

## 把一封邮件存入仓库

每封邮件上都有三个按钮：

- **保存为笔记**——在你的邮件文件夹中创建一篇笔记（`YYYY-MM-DD 主题.md`），Frontmatter中带有发件人和日期，主题标题下方是纯文本正文。把同一封邮件捕获两次，会打开已有的笔记，而不是重复创建。
- **+ .eml**——额外把原始文件存放在笔记旁边并链接它。`.eml`包含一切，包括附件，并且可以在任何邮件程序中打开。
- **→ 任务**——在你的[默认任务数据库](Tasks.md)中创建一个条目，以主题作为标题、今天的日期作为截止日期，并预先填入未完成状态。

## 把内容取出——不发送

Plainva从不使用SMTP协议。取而代之：

- **以笔记回复**（在一封邮件上）：创建一篇发给发件人的笔记（Frontmatter中的`to:`）并附带引用的原文——在Plainva中写下你的回复。
- **将笔记保存为邮箱中的草稿**（命令面板，在任意打开的笔记上）：通过IMAP把这篇笔记存为你**自己邮箱中的一份草稿**——选择账户、收件人和草稿文件夹，然后打开你平常使用的邮件程序，在那里检查并发送。格式会被保留。
- **通过邮件发送笔记（mailto）**（命令面板）：用纯文本形式在你的默认邮件程序中打开这篇笔记（长笔记会被缩短）。
- **将笔记复制为邮件文本**（命令面板）：把带格式的笔记放入剪贴板——粘贴到任意邮件撰写窗口中即可。
